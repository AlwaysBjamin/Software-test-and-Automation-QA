# Software-test-and-Automation-QA
I developed and tested the Contact Service by creating the Contact, ContactService, ContactTest, and 
ContactServiceTest classes. The project focused on validating contact information, preventing invalid data,
and ensuring contacts could be added, retrieved, updated, and deleted correctly. I ensured my code was
functional and secure by validating user input and writing JUnit tests for both valid and invalid scenarios,
confirming that the program accepted correct data while rejecting duplicate IDs, null values, and other invalid input.

I interpreted user needs by identifying the required features and implementing methods to manage 
contacts while enforcing all project requirements. I approached the software design by breaking the project
into smaller components, first defining the data and validation rules, then building the service methods, 
and finally writing tests to verify each requirement. This process resulted in code that is organized, 
maintainable, and easier to debug.
